TeaCache threshold

Skips transformer work on low-change steps. Higher skips more: faster, lower fidelity.

Choosing a threshold

The threshold bounds accumulated per-step change, so the safe value depends on sampler and steps. High step counts with single-step or secant samplers stay near-lossless up to ~0.3–0.5; few-step multistep samplers (dpmpp_2m) need 0.01 or less.

Maps the threshold through the fitted curve (Settings → TeaCache) so it tracks true per-step change. When off, the raw estimate maps differently; re-tune the threshold.

Forecast basis

How skipped steps extrapolate the cached residual. HiCache's damped Hermite basis tracks curved trajectories the linear forecast overshoots; same skip decisions, only the skip output changes. Measurably better with ancestral samplers; on plain euler TaylorSeer is marginally closer to the uncached image.

DeepCache interval

Reuses deep UNet features across steps, recomputing only the detail layers.

Choosing an interval

Interval N runs the full UNet every Nth step and reuses the cache on the others; higher is faster but loses fidelity. N=2 is a near-lossless ~1.5–1.8× speedup; 3 and up trade more quality for speed.
